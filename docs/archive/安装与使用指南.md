# 安装与使用指南

> **⚠️ 已归档** — 本文档为历史参考，部分命令和路径可能已过时。当前快速启动见 `docs/01_QUICK_START.md`。

## 1. 环境要求

| 依赖 | 版本要求 |
|------|----------|
| Python | >= 3.8 |
| Node.js | >= 18（用于前端构建） |
| Docker | >= 20.x（用于部署 Neo4j） |

| pip | 最新版 |

## 2. 安装步骤

### 2.1 克隆项目

```bash
git clone https://github.com/davis-you/MedicalGraphRAGSystem.git
cd MedicalGraphRAGSystem
```

### 2.2 创建 Python 虚拟环境

**强烈推荐**使用虚拟环境，避免与系统 Python 或其他项目冲突。

```bash
# 创建虚拟环境（在项目根目录下）
python3 -m venv venv

# 激活虚拟环境
# macOS / Linux:
source venv/bin/activate
# Windows:
# venv\Scripts\activate

# 激活后终端提示符会出现 (venv) 前缀，表示已进入虚拟环境
# 后续所有 pip / python 命令都在虚拟环境中执行
```

> **退出虚拟环境**：输入 `deactivate`。
> **每次打开新终端**都需要重新激活：`source venv/bin/activate`

### 2.3 安装 Python 依赖

确保已激活虚拟环境（终端有 `(venv)` 前缀），然后安装依赖：

```bash
pip install -r requirements.txt
```

依赖说明：

| 包名 | 用途 |
|------|------|
| python-dotenv | 环境变量管理（支持 `.env` 文件） |
| requests | 爬虫 HTTP 请求 |
| lxml | HTML 解析（XPath） |
| langchain-core | LangChain 核心框架 |
| langchain-ollama | Ollama 本地模型（默认 LLM 提供商） |
| py2neo | Neo4j 图数据库操作 |
| rapidfuzz | 实体模糊匹配（问答系统实体归一化） |
| pyahocorasick | 问句分类降级（AC 自动机，LLM 不可用时使用） |
| fastapi | Web API 框架 |
| uvicorn | ASGI 服务器（运行 FastAPI） |

**商业 LLM API（可选，按需安装）：**

```bash
# OpenAI / DeepSeek 等兼容 API
pip install langchain-openai

# Anthropic Claude
pip install langchain-anthropic
```

**Ollama 本地模型（默认，用于问答意图识别 + 并发症分词）：**

```bash
# 安装 Ollama: https://ollama.com
# 拉取模型
ollama pull qwen3:8b
# 确保 Ollama 服务在运行
ollama serve
```

> 如果 LLM 不可用，爬虫和问答系统都会自动降级（词典匹配 / 关键词规则），不影响运行。

### 2.4 安装 Docker（macOS）

如果你的 Mac 还没有安装 Docker，按以下步骤操作：

**Step 1：安装 Homebrew（已有可跳过）**

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Apple Silicon 芯片（M1/M2/M3/M4）安装后需要添加环境变量：

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

**Step 2：安装 Docker Desktop**

```bash
brew install --cask docker
```

安装完成后，从「应用程序」中打开 Docker Desktop，等待菜单栏出现 Docker 图标（鲸鱼）表示启动成功。

**Step 3：安装 Docker CLI 工具**

```bash
brew install docker docker-compose
```

**Step 4：验证安装**

```bash
docker --version
# Docker version 27.x.x, build xxxxx

docker run hello-world
# 看到 "Hello from Docker!" 即为成功
```

### 2.5 部署 Neo4j 数据库

**方式一：Docker 部署（推荐）**

```bash
docker run -d \
  --name neo4j \
  --restart unless-stopped \
  -p 7474:7474 \
  -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/your_password \
  -e NEO4J_PLUGINS='["apoc"]' \
  -v neo4j-data:/data \
  -v neo4j-logs:/logs \
  neo4j:5-community
```

启动后验证：

```bash
# 检查容器状态
docker ps | grep neo4j

# 浏览器访问管理界面
open http://localhost:7474
```

首次登录使用用户名 `neo4j`，密码为上面 `NEO4J_AUTH` 中设置的值（`your_password`）。

常用 Docker 管理命令：

```bash
docker stop neo4j        # 停止
docker start neo4j       # 启动
docker restart neo4j     # 重启
docker logs neo4j        # 查看日志
docker rm -f neo4j       # 删除容器（数据保留在 volume 中）
```

**方式二：本地安装**

从 [Neo4j 官网](https://neo4j.com/download/) 下载安装，启动后访问 `http://localhost:7474` 设置密码。

**方式三：远程服务器 Docker 部署**

如果你有远程 Linux 服务器，可以将 Neo4j 部署在服务器上：

```bash
# SSH 连接服务器
ssh root@your_server_ip

# 安装 Docker（以 CentOS 为例）
curl -fsSL https://get.docker.com | sh
systemctl enable --now docker

# 部署 Neo4j（低配服务器建议限制内存）
docker run -d \
  --name neo4j \
  --restart unless-stopped \
  -p 7474:7474 \
  -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/your_password \
  -e NEO4J_server_memory_heap_initial__size=256m \
  -e NEO4J_server_memory_heap_max__size=512m \
  -e NEO4J_server_memory_pagecache_size=128m \
  -v neo4j-data:/data \
  -v neo4j-logs:/logs \
  neo4j:5-community
```

> 注意：远程部署需确保服务器防火墙/安全组开放 7474（HTTP）和 7687（Bolt）端口。

### 2.6 配置项目（.env 文件）

项目使用统一配置文件 `settings.py`，通过环境变量或 `.env` 文件配置。

## 3. 使用流程

整个项目分四步：**爬取数据 → 构建图谱 → 启动问答 → 启动 Web 界面**。Web 界面同时提供**基础问答**和 **GraphRAG 问答**两种模式。

### 3.1 爬取数据

项目已附带预爬好的 `data/medical.json`（8800+ 条疾病记录），可直接跳到 3.2。

如需重新爬取：

```bash
cd data_spider

# 爬取全部数据（约 12-15 小时）
python3 main.py

# 小范围测试
python3 main.py --start 1 --end 10 --test

# 指定范围爬取
python3 main.py --start 1 --end 500

# 中断后恢复
python3 main.py --resume
```

爬虫参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--start` | 1 | 起始页码 |
| `--end` | 11000 | 结束页码 |
| `--output` | `data/medical.json` | 输出路径 |
| `--resume` | - | 断点续爬 |
| `--delay` | 0.3 | 请求间隔（秒） |
| `--test` | - | 测试模式，只打印不写文件 |
| `--ollama-model` | `qwen3:8b` | Ollama 模型名称 |
| `--ollama-url` | `http://localhost:11434` | Ollama 服务地址 |

### 3.2 构建知识图谱

确保 Neo4j 已启动，执行：

```bash
cd knowledge_graph

# 首次构建（清空旧数据 + 全量导入）
python3 main.py --password your_password --clear

# 远程 Neo4j
python3 main.py --uri bolt://192.168.1.10:7687 --user neo4j --password xxx --clear

# 只创建节点
python3 main.py --password xxx --step nodes

# 只创建关系（节点已存在时）
python3 main.py --password xxx --step rels
```

图谱构建参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--uri` | `bolt://127.0.0.1:7687` | Neo4j 连接地址 |
| `--user` | `neo4j` | Neo4j 用户名 |
| `--password` | `neo4j` | Neo4j 密码 |
| `--data` | `../data/medical.json` | JSONL 数据文件路径 |
| `--clear` | - | 构建前清空已有图谱 |
| `--step` | `all` | 执行步骤: `all` / `nodes` / `rels` |

构建内容：
- 7 类实体节点（疾病、药品、食物、检查项、科室、药企、症状），约 4.4 万个
- 11 类实体关系，约 30 万条
- 使用 UNWIND 批量写入 + 索引加速，耗时数分钟

### 3.5 GraphRAG 问答模式

GraphRAG 是基于知识图谱子图检索 + LLM 生成的高级问答模式，适合处理复杂、多实体的问题。

在 Web 界面顶部点击 **GraphRAG** 切换到该模式，然后即可提问：

```
试试: 糖尿病和高血压有什么共同的并发症和用药？
```

GraphRAG 管线流程：
1. LLM 提取问题中的实体（不需要固定意图分类）
2. 实体归一化到知识图谱标准名称
3. 多跳子图检索（1-2 跳邻居 + Disease 属性）
4. 子图组装为结构化文本上下文（限 6000 字符）
5. LLM 基于图谱上下文生成自然语言回答

当 Ollama 不可用时，GraphRAG 会自动降级到基础问答模式。

> 详细技术设计见 `doc/GraphRAG技术设计文档.md`。

## 5. 常见问题

**Q: 爬虫报错 `ConnectionError`**
A: 网站有频率限制，增大延迟：`--delay 1.0`。

**Q: Neo4j 连接失败**
A: 检查 Neo4j 是否启动（`http://localhost:7474`），确认用户名密码配置正确。

**Q: 导入图谱太慢**
A: 新版使用批量写入 + 索引，通常数分钟完成。如仍较慢，确认 Neo4j 内存配置（建议 heap ≥ 512M）。

**Q: 图谱重复执行会产生重复数据吗？**
A: 不会。使用 MERGE 操作，同名节点和关系只创建一次。也可用 `--clear` 清空后重建。

**Q: `pyahocorasick` 安装失败**
A: 需要 C 编译器。macOS 执行 `xcode-select --install`，Linux 执行 `apt install build-essential`。

**Q: 爬虫中断了怎么办**
A: 使用 `cd data_spider && python3 main.py --resume` 从断点继续。

**Q: 问答系统没有 LLM 能用吗？**
A: 可以。系统有三级降级策略，Ollama 不可用时会自动退回到词典 NER + 关键词规则模式。

**Q: `rapidfuzz` 安装失败**
A: 需要 C 编译器。macOS 执行 `xcode-select --install`，或使用 `pip3 install rapidfuzz --only-binary :all:`。

**Q: Web 界面打开后请求失败**
A: 确认后端已启动（`python3 -m server.app`），开发模式下确认 Vite dev server 在运行且端口正确。

**Q: `npm run build` 失败**
A: 确保安装了 devDependencies：`npm install --include=dev`。Node.js 版本需 >= 18。

**Q: 如何使用 OpenAI / Claude 替代 Ollama？**
A: 安装对应包后修改 `.env`。例如 OpenAI：`pip3 install langchain-openai`，然后设置 `LLM_PROVIDER=openai`、`LLM_MODEL=gpt-4o`、`OPENAI_API_KEY=sk-...`。详见 §2.6。

**Q: 如何使用 DeepSeek API？**
A: DeepSeek 兼容 OpenAI 协议。设置 `LLM_PROVIDER=openai`、`LLM_MODEL=deepseek-chat`、`OPENAI_API_KEY=your_key`、`OPENAI_BASE_URL=https://api.deepseek.com/v1`。
