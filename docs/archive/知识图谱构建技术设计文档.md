# 医药知识图谱构建 — 技术设计文档

> **⚠️ 已归档** — 本文档为历史参考。当前图谱构建见 `knowledge_graph/` 和 `docs/06_DATABASE_SCHEMA.md`。

## 1. 概述

本模块从爬虫产出的 JSONL 数据文件（`data/medical.json`）读取疾病结构化信息，构建以疾病为中心的医药知识图谱并写入 Neo4j 图数据库，供下游问答系统查询使用。

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| 模块化 | 配置、数据加载、图谱写入、入口分离为独立文件 |
| 高性能 | UNWIND 批量写入（500 条/批），较原版逐条插入提升数十倍 |
| 幂等 | 使用 MERGE 代替 CREATE，重复执行不会产生重复数据 |
| 安全 | 参数化 Cypher 查询，消除注入风险 |
| 可配置 | Neo4j 地址、用户名、密码、数据路径均支持 CLI 参数 |

### 1.2 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| 图数据库 | Neo4j 5.x Community | Bolt 协议通信 |
| Python 驱动 | `py2neo >= 2021.2.3` | 封装 Cypher 执行与事务管理 |
| 数据格式 | JSONL | 每行一个 JSON 对象，流式读取 |

## 3. 图谱 Schema

### 3.1 节点类型（7 类）

| 标签 | 含义 | 数量级 | 属性 |
|------|------|--------|------|
| Disease | 疾病 | ~8,800 | name, desc, prevent, cause, easy_get, cure_department, cure_way, cure_lasttime, cured_prob, get_prob, yibao_status, get_way, cost_money |
| Drug | 药品 | ~3,500 | name |
| Food | 食物 | ~1,500 | name |
| Check | 检查项 | ~3,300 | name |
| Department | 科室 | ~60 | name |
| Producer | 药品厂商 | ~17,000 | name |
| Symptom | 症状 | ~5,900 | name |

**总计约 4.4 万个节点。**

### 3.2 关系类型（11 类）

| 关系类型 | 起始节点 | 目标节点 | 中文名 | 说明 |
|----------|----------|----------|--------|------|
| `has_symptom` | Disease | Symptom | 症状 | 疾病有哪些症状 |
| `acompany_with` | Disease | Disease | 并发症 | 疾病之间的并发关系 |
| `belongs_to` | Disease | Department | 所属科室 | 疾病归属哪个科室 |
| `dept_belongs_to` | Department | Department | 属于 | 科室层级（小科室→大科室） |
| `common_drug` | Disease | Drug | 常用药品 | 疾病的常用药品 |
| `recommand_drug` | Disease | Drug | 好评药品 | 疾病的推荐药品 |
| `do_eat` | Disease | Food | 宜吃 | 患病时宜吃的食物 |
| `no_eat` | Disease | Food | 忌吃 | 患病时忌吃的食物 |
| `recommand_eat` | Disease | Food | 推荐食谱 | 患病时的推荐食谱 |
| `need_check` | Disease | Check | 诊断检查 | 疾病需要做的检查项 |
| `drugs_of` | Producer | Drug | 生产药品 | 厂商生产哪些药品 |

**总计约 30 万条关系。**

### 3.3 图谱关系示意

```
                    ┌──────────┐
                    │Department│
                    └────▲─────┘
            belongs_to   │  dept_belongs_to
                    ┌────┴─────┐
  ┌───────┐         │          │         ┌────────┐
  │Symptom│◄────────┤ Disease  ├────────►│  Check │
  └───────┘         │          │         └────────┘
   has_symptom      │          │ need_check
                    ├──────────┤
        do_eat /    │          │  common_drug /
        no_eat /    │          │  recommand_drug
     recommand_eat  │          │
                    ▼          ▼
              ┌──────┐   ┌──────┐   ┌────────┐
              │ Food │   │ Drug │◄──│Producer│
              └──────┘   └──────┘   └────────┘
                           drugs_of
```

## 4. 模块详细设计

### 4.1 config.py — 配置中心

**职责**: 集中管理路径、Neo4j 连接参数和图谱 Schema 定义。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `NEO4J_URI` | `bolt://127.0.0.1:7687` | Neo4j Bolt 协议地址 |
| `NEO4J_USER` | `neo4j` | 用户名 |
| `NEO4J_PASSWORD` | `neo4j` | 密码 |
| `DATA_PATH` | `../data/medical.json` | 输入数据文件 |
| `NODE_LABELS` | 7 个标签 | 节点类型定义 |
| `REL_TYPES` | 11 个关系 | 关系类型及起止节点映射 |
| `DISEASE_PROPS` | 13 个字段 | Disease 节点携带的属性列表 |

### 4.2 data_loader.py — 数据加载器

**职责**: 一次性读取 JSONL，提取全部节点集合和关系列表。

**输出结构**:

```python
nodes = {
    "Disease":    set(...),   # 疾病名称集合
    "Drug":       set(...),   # 药品名称集合
    "Food":       set(...),   # 食物名称集合
    "Check":      set(...),   # 检查项集合
    "Department": set(...),   # 科室集合
    "Producer":   set(...),   # 厂商集合
    "Symptom":    set(...),   # 症状集合
}

rels = {
    "has_symptom":    [(src, dst), ...],
    "acompany_with":  [(src, dst), ...],
    ...  # 共 11 类
}

disease_infos = [
    {"name": "...", "desc": "...", "cause": "...", ...},
    ...  # 每个疾病的属性字典
]
```

**关键处理逻辑**:

| 字段 | 处理方式 |
|------|----------|
| `cure_department` | 长度 1 → 直接归属；长度 ≥ 2 → 取第二个归属 + 建立科室层级关系 |
| `drug_detail` | 按 `(` 拆分厂商名和药品通用名，建立 Producer → Drug 关系 |
| 列表类属性 | 转为逗号分隔字符串存入 Disease 节点属性 |
| 所有关系 | 自动去重（`set` 去重） |

### 4.3 graph_builder.py — 图谱构建器

**职责**: 封装所有 Neo4j 写入操作。

#### 4.3.1 索引创建

为每个节点标签的 `name` 属性创建索引，加速关系匹配时的 `MATCH` 查询：

```cypher
CREATE INDEX IF NOT EXISTS FOR (n:Disease) ON (n.name)
CREATE INDEX IF NOT EXISTS FOR (n:Drug) ON (n.name)
-- ... 共 7 个索引
```

#### 4.3.2 批量节点创建

**简单节点**（Drug / Food / Check 等，只有 `name` 属性）:

```cypher
UNWIND $names AS n MERGE (:Drug {name: n})
```

**疾病节点**（13 个属性）:

```cypher
UNWIND $batch AS d
MERGE (n:Disease {name: d.name})
SET n.desc = d.desc, n.prevent = d.prevent, ...
```

#### 4.3.3 批量关系创建

```cypher
UNWIND $batch AS r
MATCH (p:Disease {name: r.src})
MATCH (q:Symptom {name: r.dst})
MERGE (p)-[rel:has_symptom {name: $rel_name}]->(q)
```

**关键设计决策**:

| 决策 | 原因 |
|------|------|
| `MERGE` 而非 `CREATE` | 幂等，重复执行不产生重复数据 |
| `UNWIND` 批量写入 | 减少网络往返，500 条/批 |
| 参数化查询（`$batch`） | 防止 Cypher 注入，值不拼入语句 |
| 先创建索引再写关系 | 索引加速 `MATCH` 节点查找 |

### 4.4 main.py — 入口

**职责**: 解析 CLI 参数，串联加载 → 连接 → 建索引 → 建节点 → 建关系全流程。

**执行流程**:

```
1. 解析 CLI 参数（--uri, --user, --password, --data, --clear, --step）
2. DataLoader.load()          → 读取 JSONL，提取节点和关系
3. GraphBuilder(uri, user, pw) → 连接 Neo4j
4. [可选] clear_all()          → 清空旧数据
5. create_indexes()            → 7 个索引
6. build_nodes()               → Disease(~8800) + 6 类简单节点
7. build_rels()                → 11 类关系(~30 万)
8. 输出总耗时
```

## 5. 与原版对比

| 对比项 | 原版 `build_medicalgraph.py` | 新版 `knowledge_graph/` |
|--------|------------------------------|-------------------------|
| 代码结构 | 单文件 275 行 | 4 个文件，职责分离 |
| 写入方式 | 逐条 `g.create(Node)` / 逐条 Cypher | `UNWIND` 批量 500 条/批 |
| 幂等性 | `CREATE` 重跑产生重复 | `MERGE` 幂等可重跑 |
| Cypher 安全 | 字符串格式化拼接（`%s`） | 参数化查询（`$param`） |
| 数据加载 | `read_nodes()` 被调用 2 次 | 只加载 1 次 |
| 索引 | 无 | 自动创建 7 个 name 索引 |
| 连接配置 | 硬编码在代码中 | CLI 参数可配置 |
| 日志 | `print()` 输出 | `logging` 结构化日志 |
| 分步执行 | 不支持 | `--step nodes/rels` 可单独执行 |
| 清空图谱 | 不支持 | `--clear` 参数 |
| 预估耗时 | 1-2 小时 | 数分钟（批量写入 + 索引加速） |

## 6. 使用方式

### 6.1 CLI 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--uri` | str | `bolt://127.0.0.1:7687` | Neo4j 连接地址 |
| `--user` | str | `neo4j` | Neo4j 用户名 |
| `--password` | str | `neo4j` | Neo4j 密码 |
| `--data` | str | `../data/medical.json` | JSONL 数据文件路径 |
| `--clear` | flag | false | 构建前清空已有图谱 |
| `--step` | str | `all` | 执行步骤: `all` / `nodes` / `rels` |

### 6.2 运行示例

```bash
cd knowledge_graph

# 默认参数（本地 Neo4j）
python3 main.py --password your_password

# 远程 Neo4j + 清空旧数据
python3 main.py --uri bolt://192.168.1.10:7687 --user neo4j --password xxx --clear

# 只创建节点（调试用）
python3 main.py --password xxx --step nodes

# 只创建关系（节点已存在时）
python3 main.py --password xxx --step rels

# 指定数据文件
python3 main.py --data /path/to/medical.json --password xxx
```

## 8. Neo4j 验证查询

图谱构建完成后，可在 Neo4j Browser（`http://localhost:7474`）中执行以下 Cypher 验证：

```cypher
-- 统计各类节点数量
MATCH (n) RETURN labels(n)[0] AS label, count(*) AS count ORDER BY count DESC

-- 统计各类关系数量
MATCH ()-[r]->() RETURN type(r) AS rel_type, count(*) AS count ORDER BY count DESC

-- 查询某个疾病的完整关联
MATCH (d:Disease {name: '糖尿病'})-[r]->(n)
RETURN d.name, type(r), n.name LIMIT 50

-- 查看科室层级
MATCH (d1:Department)-[:dept_belongs_to]->(d2:Department)
RETURN d1.name AS 子科室, d2.name AS 上级科室

-- 查找某药品治疗哪些疾病
MATCH (d:Disease)-[:common_drug|recommand_drug]->(drug:Drug {name: '阿莫西林'})
RETURN d.name AS 疾病
```
