# 项目规则

## 技术栈
- Python 3.10+ / LangChain 1.x / LangGraph 1.x
- Neo4j 5.x (py2neo) / FastAPI + Uvicorn
- React 19 + TypeScript + Vite 8 + Tailwind CSS 4 + shadcn/ui

## 硬性规则

### 架构
- DDD 分层：表现层(frontend/) → 网关层(backend/api/) → 领域层(backend/domain/) → 数据层(Neo4j + backend/dict/)
- 所有 LLM 调用通过 `backend/core/config.py` 的 `create_llm()` 工厂函数获取模型
- 禁止使用 `from langchain import ...` 或已弃用的 `Chain`/`Agent`
- LangGraph 工作流节点函数保持纯函数风格

### TDD
- 必须 RED → GREEN → REFACTOR，不得跳过任何阶段
- 每个 Story 完成后必须 Commit
- 不得实现超出当前 Story 范围的功能

### Contract First
- API 契约定义在 `docs/04_CONTRACT.md`，是接口的唯一权威来源
- 先定义接口，后实现代码
- 实现必须通过 Contract Test

### 代码风格

**Python**
- 遵循 PEP 8，4 空格缩进（见 `.editorconfig`）
- 行宽 ≤ 100 字符（与 ruff/pyproject.toml 保持一致）
- 导入顺序：标准库 → 第三方 → 本地模块
- 类型注解：函数签名必须标注参数和返回值类型
- 文档字符串：Google 风格 docstring
- 命名：模块/包 `snake_case`，类 `PascalCase`，函数/变量 `snake_case`，常量 `UPPER_SNAKE_CASE`

**TypeScript / React**
- 2 空格缩进（见 `.editorconfig`）
- 组件 `PascalCase`，文件名与组件名一致
- Hooks `camelCase`，以 `use` 开头
- 类型/接口 `PascalCase`，优先使用 `interface`
- 使用函数组件 + Hooks，禁止 class 组件

**通用**
- 编码 UTF-8，换行符 LF（`\n`）
- 文件末尾保留一个空行，去除行尾空白字符
- 注释和文档使用中文

### Git 提交规范（企业 Conventional Commits）

**格式**

```
<type>(<scope>): <中文简述>

<body>（可选，说明 why 而非 what）

<footer>（可选，BREAKING CHANGE / 关联 Issue）
```

**Type 类型**

| type | 含义 | 示例 |
|------|------|------|
| feat | 新功能 | feat(qa_engine): 新增三级降级分析路由 |
| fix | 修复 bug | fix(api): SSE 流式响应偶发断连增加心跳 |
| test | 测试相关 | test: 新增多轮对话集成测试 4 个用例 |
| refactor | 重构 | refactor(kbqa): 移除硬编码 Cypher 模板 |
| docs | 文档 | docs: 更新架构文档后端路径引用 |
| style | 格式/样式 | style(frontend): 统一聊天面板间距 |
| chore | 杂项/工具 | chore: 添加 .gitignore data/ 规则 |
| perf | 性能优化 | perf(graphrag): 子图检索批量查询替代逐条查询 |

**Scope 范围**

| scope | 对应目录 | 说明 |
|-------|----------|------|
| api | backend/api/ | FastAPI 服务、SSE、路由、模型 |
| core | backend/core/ | 配置、CLI 入口、LLM 工厂 |
| qa_engine | backend/domain/qa_engine/ | LangGraph 工作流、节点、流式 |
| kbqa | backend/domain/kbqa/ | 模板问答、Cypher 生成、实体归一化 |
| graphrag | backend/domain/graphrag/ | 子图检索、实体抽取、上下文构建 |
| knowledge_graph | backend/domain/knowledge_graph/ | 图谱构建、数据加载 |
| data_spider | backend/domain/data_spider/ | 数据采集、分词 |
| frontend | frontend/ | React 前端、UI 组件 |
| docs | docs/ | 项目文档 |
| docker | Dockerfile, docker-compose.yml | 容器化部署 |
| deps | pyproject.toml | 依赖管理 |
| tests | backend/tests/ | 测试文件 |

**规范**

- subject 行不超过 72 字符，中文描述
- body 说明**为什么**改，而非改了什么（代码 diff 已经说明了 what）
- 每次 commit 必须通过全部测试
- 不同功能分开 commit，不混在一起
- 不提交 .env、密钥、缓存文件

**Footer 规范**

| Footer 类型 | 格式 | 何时使用 |
|-------------|------|----------|
| 破坏性变更 | `BREAKING CHANGE: <描述>` | API/数据结构不兼容变更 |
| 关联 Issue | `Closes #123` | 修复或实现某个 Issue |
| 组合使用 | 两者可同时出现 | 破坏性变更 + 关联 Issue |

**完整示例**

```
feat(api): SSE 流式响应新增阶段性进度事件

原 stream_qa 只返回 delta 和 done 两种事件，
前端无法感知当前处于模板检索还是 GraphRAG 路径。
现新增 status 事件 {stage, message}，
前端可据此展示路由标签和进度提示。

BREAKING CHANGE: /api/chat/stream SSE 事件类型新增 status，
旧版前端需同步更新事件解析逻辑
```

```
fix(qa_engine): 模板无结果时回退 GraphRAG 路径失效

模板路径 execute_query 返回空结果时，
condition 边判断逻辑有误导致直接走 error_handler，
而非预期的 template→graphrag 回退。

Closes #42
```

```
refactor(kbqa): 实体归一化器改用 rapidfuzz 替代 difflib

difflib 在 4.4 万实体词典上模糊匹配耗时 2-3s，
rapidfuzz 降至 200ms 以内，且准确率更高。
```

**分支策略**

- `main` — 稳定分支，随时可部署
- `feature/<功能名>` — 新功能开发
- `fix/<问题描述>` — 紧急修复

### 迭代优化规则
- 优化不得影响现有系统正常运行，改动前必须确认测试 baseline 全绿
- 每个阶段在独立 feature 分支开发，人类 Review 通过后才合并到 main
- 改动较大时使用影子模式（新旧并存 + 配置切换），人类确认后再切换
- 详细计划见 `docs/09_ITERATION_PLAN.md`，各阶段 Story 级文档在 `docs/iteration/`

### 安全
- 严禁提交 `.env`、密钥文件、凭证到仓库
- `.env.example` 提供模板，不含真实密钥
- 敏感配置一律通过环境变量注入
